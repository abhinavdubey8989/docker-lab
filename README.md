
This repo contains experiment with various tools using docker (mostly).


# Conventions
- As a convention, the `_volume` dir is used to store any data produced by the container
    - ie. type of docker volume is : `bind-mount`
    - This data can be later used to inspect or observe any behaviour
    - Eg: in `prometheus-server`, the data generated by container is stored at `./_volume/prometheus`

- If any data is to be supplied from the host to docker, it is done via dedicated dir
    - ie. type of docker volume is : `bind-mount` (again)
    - dir name is same/similar to container name
    - Eg: in `statsd-grafana-graphite`, statsd config is supplied from Host machine
    - This config is present in `./statsd/config.js` & supplied in the volumn of statsd service
